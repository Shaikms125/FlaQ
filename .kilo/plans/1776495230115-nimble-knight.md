# Refactor: AI Quiz App — Chat Interface & Dashboard Redesign

## Overview

Replace the current popup/dialog-based quiz generation with a chat-like interface embedded in a redesigned Home dashboard using Shadcn Tabs. Extract a universal `ShareQuiz` component for reuse in both the chat success state and the My Quizzes page.

---

## Architecture Changes

### Current Flow
- Home page is empty (welcome text only)
- Sidebar "Create Quiz" button → opens `GenerateQuizDialog` (modal, 3-step wizard)
- `ShareDialog` is tightly coupled to `QuizCard`

### Target Flow
- Home page → Shadcn `Tabs` with "Take a Test" and "Generate Quiz"
- "Generate Quiz" tab → Chat-like interface (prompt input at bottom, scrolling messages)
- After quiz creation → Success state with Public/Unlimited/Share actions
- Universal `ShareQuiz` component used in both post-creation and My Quizzes

### New Files (4)

#### 1. `src/components/quiz/ShareQuiz.tsx` — Universal Share Component
- Uses Shadcn `Dialog` (consistent with existing `ShareDialog` pattern)
- Builds on the existing `ShareDialog` design but makes it more robust
- Props: `quizId: Id<"quizzes">`, `quizTitle: string`, `accessCode: string`, `isPublic: boolean`, `allowUnlimitedAttempts: boolean`, `availableFrom?: number`, `availableTo?: number`, `trigger?: React.ReactElement`

  - **Copy share link** (`/quiz/{accessCode}`)

  - **Social share buttons** (Twitter/X intent, WhatsApp, email via `mailto:` — no external SDKs)

#### 2. `src/components/quiz/ChatInterface.tsx` — Main Chat UI
- Full-height chat container with:
  - **Top bar**: Title + "Manual Quiz" `Switch` toggle (top-right)
  - **Settings bar** (below top bar): 3 Shadcn `Select` components

    - Question count (default: 10, range: 5-30 in steps of 5)

  - **Input area** (bottom): `Input` + send `Button`, disabled during loading
- Chat message types:
  - User message (prompt text, right-aligned)
  - AI message (left-aligned, shows generation progress then results)
  - System message (centered, e.g., "Quiz created successfully!")
- Uses `useAction(api.quiz.generateQuiz)` and `useMutation(api.quiz.saveQuiz)` — same backend calls as current dialog
- State machine: `idle` → `generating` → `preview` → `saving` → `success`

#### 3. `src/components/quiz/ChatMessage.tsx` — Individual Chat Bubble

#### 4. `src/components/quiz/QuizCreationSuccess.tsx` — Post-Creation Actions

## "Take a Test" Tab Design

The "Take a Test" tab should be a **quick, simple form** (NOT chat-based) — the goal is speed:
- Prompt textarea ("What do you want to be quizzed on?")
- Quick settings: question count select, time limit select
- "Start Test" button
- This reuses the existing `handleTakeTest` logic from `GenerateQuizDialog`:
  - Generates quiz via AI
  - Auto-saves as private with unlimited attempts
  - Immediately navigates to `/quiz/{accessCode}` to start taking it

## Key Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Chat AI loading | Animated typing dots (3 bouncing dots) as AI message | Matches chat UX conventions; makes wait feel interactive |
| Manual quiz in chat | When "Manual Quiz" switch is ON, chat becomes a form-like interface with title input + create button | Keeps everything in one tab; no separate page needed |
| Preview in chat | After AI generates, show editable question cards inline in chat | Consistent with chat UX; avoids modal-within-chat |
| ShareQuiz component | Uses Shadcn Dialog (not Popover) | Consistent with existing ShareDialog; more room for content |
| Social sharing | Simple `window.open()` links (Twitter intent, WhatsApp, mailto) | No external SDK dependencies; lightweight |
| State management | Local useState within ChatInterface + Zustand for tab selection | Minimal complexity; matches existing patterns |
